## 特征
- 特征1： 支持OOP
    - 数据封装
    - 数据抽象
    - 类型继承
    - 多态
- 特征2：强静态类型
    - 所有变量都必须声明和定义
    - 所有的UDT都必须完整定义
    - 提供C++安全casting
    - 提供有限的运行态辨识服务
- 特征3：UDT是一等公民
    - UDT实例可以作为值使用
    - UDT实例可以在任何内存域中实现
        - stack
        - heap
        - 全程变量区
- 特征4：类型清除
    - 变量绑定
    - 操作绑定
    - 类型推论
- 特征5：支持模版
    - 隐式调用
    - 显示调用
- 特征6：指针和REF
    - 指针
    - REF
    - 多态变量 （指针类型转换）
- 特征7：类型修正符
    - const 、mutable
        - 编译阶段生效
    - volatile
    - explicat 显示调用
    - 类型实例的自动转换

## 类型系统作用
- 1.程序的精确性保障
    - 程序严谨性
    - 错误在编译阶段提前发现
    - runtime 的类型异常处理机制
- 2.性能的可靠保障
    - 静态的语义解释
        - 编译时间换运行时间
        - 目标代码空间换运行时间
    - 静态优化
    - 类型清除
